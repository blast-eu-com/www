REALM

The realm is a Blast object used to control the user permissions of all others Blast object. Every Blast objects are assigned to the active user realm at the object creation time. Once an object is created, it can be shared to several realms.

A realm has an unlimited number of members. By default any member has a regular role expect the creator of the realm and his delegates. The owner role is assigned to the creator of the realm at the creation time, then the creator can elect up to three delegates. The content of a realm can be accessible by any of his members only.

  • regular has read/write/execute access.
  • delegate same as regular role and can approve new member request.
  • owner same as delegate role and can grant/revoke delegate role or owner role to anyone.
If a user wants to become a new member then a request must be sent, the owner or one of his delegates must acknowledge the request before this new member can start interacting with the content of the realm. As all realm's objects are linked to the realm then a member can leave a realm at any moment. If the owner decides to leave a realm, he must before elect a new owner, either he can grant one of his delegates or anyone of the regular members.

Realm form

A new cluster can be created by filling the cluster form. Every newly created cluster will be part of the current realm.

  • cluster name is a mandatory field, it names the realm object.
  • cluster description is an optional field, it describes the realm object shortly.

Realm list

All clusters of the current realm are visibile as card in the list section. Click on the cluster card to access to the cluster details section.

Realm details

Click on the cluster card of your choice to open the cluster details. The cluster details contains three sections:

  • cluster details shows the id, name, description, realms.
  • cluster nodes management allows you to manage the cluster nodes.
  • cluster nodes list lists the nodes linked.
