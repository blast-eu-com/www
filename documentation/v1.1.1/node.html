NODE

The node is a Blast object member of the cluster sublayer, the node can have one more uplinks to cluster objects.

Node form

A new node can be created by filling the node form. Every newly created node will be part of the current realm.

  • node hostname is a mandatory field, it is the hostname of the node or any alias.
  • node ip is a mandatory field, it is the ip of the node.
  • node cluster is an optional field, select a parent cluster to link.
  • node description is an optional field, it describes the node object shortly.
  • scan by node ip is mandatory selector, the default value is false. It describes what parameter BLAST will use to access the node. If selected then BLAST will use the node ip else BLAST will use the node hostname.

Node list

All nodes of the current realm are visibile as card in the list section. Click on the node card to access to the node details section.

Node details

Click on the node card of your choice to access his details. The node details contains three sections:

  • node details shows the id, hostname, ip, description, mode, realms.
  • node mode allows you to manage the cluster nodes.
  • node peers lists the nodes linked.

Delete a node

Click on the node card of your choice to access his details. The node details contains the delete button to be used to delete the selected node. A node can be deleted if the node is not anymore linked to any cluster and if the node is not part of a registered scenario else the node deletion will be rejected.

Rescan a node

Click on the node card of your choice to acces his details. The node details contains the rescan button to be used to rescan the node. For a rescan to be successfull, the ssh connection details and the reference to the node must be persistent with what it was at the first scan, or it must be updated accordingly to any possible update made since the previous successfull scan.

node mode

The node mode define the mode of the node. there are two possible mode:

  • running is the default mode, if this mode is selected then the node will be kept by any running scenario.
  • maintenance is the mode to select if the node will be skipped by any running scenario.

Click on the node card of your choice to acces his details. The node details contains the node mode currently assigned to the selected node. The node mode switcher section is also accessible if a node mode switch is needed. Select the node mode to be assigned then click on the switch button.
