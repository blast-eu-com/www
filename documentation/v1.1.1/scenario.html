SCENARIO

The scenario is a Blast object. It is defined by a group of Blast objects, the scripts must be executed on the nodes in sequence or in parallel as configured by the user either for the on-demand execution or by the scenario saved.

Scenario form

A new scenario can be created by filling the scenario form. Every newly created scenario will be part of the current realm.

  • scenario name is a mandatory field, it names the scenario object.
  • scenario description is an optional field, it describes the scenario object shortly.
  • parallel mode is an optional field, the default value.
  • max parallel threads is an optional field, the default value.
  • scripts is mandatory selection, they are the scripts to be execute among the infrastructure.
  • infrastructure element is mandatory selection, they are the running script's target nodes.

Scenario script

The scripts selector is part of the scenario form, it allows to filter scripts by script name and script language. One or more scripts can be selected as scenario component.

Scenario infrastructure tree

The infrastructure selector is part of the scenario form, it offers flexible infrastructure visualization and quick select of any infrastructure, cluster and node. One or more infrastructure component can be selected as scenario component.

Save or Run a scenario

  • Save a scenario

  • A scenario can be saved to be reused several time through one or many scheduler. In order to save a scenario, the scenario form must be filled, once all parameters are reflecting the desired scenario configuration then click on the save button.

  • Run a scenario

  • An unsaved scenario can be run on-demand, the on-demand run allow one execution of a desired scenario defined in the scenario form. This scenario will not be available to define a scheduler. The run of a scenario is useful when no recurrent run is expected. In order to run a scenario, the scenario form must be filled, once all parameters are reflecting the desired configuration then click on the run button.
